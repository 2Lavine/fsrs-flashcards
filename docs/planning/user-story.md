# User Stories

## Core Workflow

### US-01 Import Cards
As a learner, I want to paste JSON flashcards generated by Claude and import them into my review queue, so I can start studying immediately.

- Given I have a JSON with deck name, source, and cards array
- When I paste it into the Import modal and click Import
- Then all cards appear in my due queue and are persisted across sessions

### US-02 Daily Review
As a learner, I want to review due flashcards each day with the FSRS algorithm scheduling my next review, so I remember what I learn long-term.

- Given I have due cards in my queue
- When I press Space to reveal the answer, then press 1/2/3/4 to rate
- Then the card is rescheduled based on FSRS and the next card appears

### US-03 Category Filtering
As a learner with cards across multiple categories, I want to filter my review queue by category, so I can focus on one topic at a time.

- Given I have cards in "数据流" and "技术栈" categories
- When I click the "数据流" chip
- Then only cards in that category appear for review

### US-04 Pause Category
As a learner, I want to temporarily skip a category during review, so cards from that topic don't distract me.

- Given I'm reviewing mixed categories
- When I click the pause toggle on a category chip
- Then cards from that category are excluded from the due queue

### US-05 Undo Rating
As a learner, I want to undo a mistaken rating, so I can correct accidental key presses.

- Given I just rated a card by accident
- When I press Ctrl+Z or A, or click the undo arrow
- Then the card returns to its previous state and I can re-rate it

### US-06 Delete Card
As a learner, I want to remove low-quality cards, so my deck stays relevant.

- Given a card is displayed
- When I press D or click Delete
- Then the card and its review history are permanently removed

### US-07 Browse & Search
As a learner, I want to browse all my cards and search by keyword, so I can find specific cards to review or verify.

- Given I have multiple decks
- When I search by question text, tag, or deck name
- Then matching cards are displayed with their current FSRS state

### US-08 Stats Dashboard
As a learner, I want to see my review activity and card distribution, so I can track my learning progress.

- Given I have review history
- When I open the Stats page
- Then I see daily review counts, streak, category breakdown, rating distribution, and FSRS parameters

### US-09 Export Cards
As a learner, I want to export my cards as JSON, so I can back them up or share with others.

- Given I have cards in my database
- When I click Export
- Then a JSON file is downloaded containing all decks and cards

### US-10 Auto-Import (Dev)
As a developer, I want the dev server to auto-import JSON files on startup, so I can seed the app without manual import.

- Given JSON files in `public/auto-import/`
- When the app loads in dev mode
- Then cards are automatically imported and available for review
